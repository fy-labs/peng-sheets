# PengSheets マニュアルテスト計画

SPECS.md と直近のバグ傾向を元に、操作×データパターンを網羅。
全チェック完了 = リリース品質OK と判断可能。

---

## テストデータパターン

テストは以下の **4つのワークブック構造** で実施する:

| ID | 構造 | ファイル例 |
|----|------|-----------|
| **W1** | シート1枚のみ | `# Tables\n## Sheet 1\n### Table 1\n\| A \| B \|...` |
| **W2** | シート複数枚 | W1 + `## Sheet 2`, `## Sheet 3` |
| **W3** | シート + ドキュメント (Multi-H1) | `# Intro\n...\n# Tables\n## S1\n...\n# Appendix\n...` |
| **W4** | 空ファイル (Onboarding) | 空ファイルまたは `# Tables` なし |

## 実施推奨順序

1. **Onboarding** (§1) — 最初の起動体験
2. **セル編集 & Undo** (§6, §10) — 基本操作の安定性
3. **シート/ドキュメント操作** (§2, §3) — 直近バグ多発領域
4. **タブ名 i18n** (§4) — 今回の修正対象
5. **タブ並び替え** (§5) — 最も複雑な領域
6. **構造操作** (§7) — 行列の挿入/削除
7. **クリップボード** (§9) — ペースト拡張
8. **保存 & 整合性** (§12) — 最終確認
